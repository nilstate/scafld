# Documentation site

scafld's documentation site is built with [Sourcey](https://sourcey.com/).
The site configuration lives in [`docs/sourcey.config.ts`](./sourcey.config.ts)
and uses two first-party Sourcey adapters:

- `markdown(...)` renders the curated guides in this `docs/` directory.
- `godoc(...)` renders the public Go API from the scafld module.

## Local preview

From the `docs/` directory:

```bash
npm ci
npm run site:dev
```

To produce a static site artefact instead:

```bash
npm ci
npm run site:build
```

The build command writes the generated site to `../dist` (relative to `docs/`).

## Maintaining navigation

Navigation is deliberately explicit. When a new Markdown guide should be
published, add its page stem to the appropriate group in
`docs/sourcey.config.ts`. For example, this page is published by adding
`"sourcey"` to the `Advanced` group. This prevents useful documentation from
existing only as a raw repository file.

## Editing pages

The configuration sets `editBranch: "main"` and `editBasePath: "docs"`, so
Sourcey links readers to the correct source location for edits. Keep guide
content in Markdown and keep the config limited to navigation, theme, and API
reference settings.

## Verification

Before submitting documentation changes, run the site build from `docs/` and
confirm that the generated output contains the new page and navigation entry.
This page documents the repository's existing Sourcey integration; it does not
claim to be auto-generated by Sourcey.
